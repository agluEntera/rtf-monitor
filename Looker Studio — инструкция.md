## 2. Структура дашборда

Рекомендую 3 страницы.

---

### Страница 1: Текущее состояние

**Источник:** `RFT — Текущие задачи`

#### Блок 1 — Сводные карточки (Scorecard)
Добавь 3 карточки рядом:

| Карточка | Поле | Фильтр |
|----------|------|--------|
| Всего задач | COUNT(issue_key) | — |
| Просрочено | COUNT(issue_key) | status_flag = Просрочена |
| On-time % | COUNT / COUNT * 100 | — |

Для On-time % используй **Calculated Field**:
```
COUNT_IF(status_flag = "В норме") / COUNT(issue_key) * 100
```

#### Блок 2 — Таблица задач
Тип: **Таблица**

Столбцы:
- `issue_key` (кликабельная ссылка: включи гиперссылку на `jira_url`)
- `status` 
- `business_days` (отсортировать по убыванию)
- `status_flag` (цветовая шкала: Просрочена — красный, В норме — зелёный)
- `developer`
- `story_points`

Фильтр страницы: элемент управления **Выпадающий список** → поле `status`
(позволит переключаться между Ready for Testing / Ready for Review / In Testing)

---

### Страница 2: Аналитика по разработчикам

**Источник:** `RFT — Текущие задачи`

#### Столбчатая диаграмма — задачи по разработчикам
- Ось X: `developer`
- Ось Y: COUNT(issue_key)
- Разбивка: `status_flag` (зелёный / красный)
- Сортировка: по убыванию

#### Таблица — детали по разработчику
- `developer`, COUNT(issue_key), SUM(story_points), COUNT просроченных
- Добавь условное форматирование: красный если просроченных > 0

---

### Страница 3: История и тренды

**Источник:** `RFT — История`

#### Линейный график — тренд времени в статусе
- Ось X: `issue_key` (или дата, если добавишь в SQL)
- Ось Y: `business_days`
- Разбивка: `status_name`
- Добавь горизонтальную линию-референс: 4 дня (Reference Line)

#### Столбчатая диаграмма — распределение по норме
- Ось X: `status_name`
- Ось Y: COUNT(issue_key)
- Разбивка: `compliance` (В норме / Просрочена)

#### Scorecard — P70 по каждому статусу
Calculated Field для каждого статуса:
```
PERCENTILE(business_days, 0.7)
```
Добавь фильтр `status_name = "Ready for Testing"` и т.д. для каждой карточки.

## 3. Автообновление

Looker Studio по умолчанию кэширует данные на 12 часов.

Чтобы данные обновлялись чаще:
- Открой **Ресурс → Управление добавленными источниками данных**
- Найди источник → **Изменить** → **Обновление данных** → выбери **1 час** или **12 часов**
